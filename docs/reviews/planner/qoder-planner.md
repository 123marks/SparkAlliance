# Qoder CLI — 星火规划模块头脑风暴指令

## 任务背景

星火校园（Spark Alliance）是面向大学生及青年群体的综合平台，技术栈为 Vue 3 + TypeScript + Vite + Supabase，整体风格为暗黑系 + 玻璃拟态 + 星火配色。

**新模块：星火规划（目标与任务）**
定位为"游戏化成长"模块，提供日/周/月计划管理、AI任务规划、社交激励、星空领域视觉激励。与已有的智能日程（Schedule）不同——日程是时间管理，规划是目标达成。

参考文档：
- `docs/project/SparkAlliance.md` 功能五（了解初始设想）
- `docs/project/MODULE-SCHEDULE.md`（了解日程模块，避免功能重叠）

## 你的任务

从**产品定位与功能架构**角度进行头脑风暴：

### 1. 与智能日程的边界划分
- 智能日程已有：时间轴安排、提醒、AI识别导入、课程表
- 星火规划应该做什么日程没做的？（目标管理？习惯打卡？里程碑？OKR？）
- 两个模块如何联动而不重叠？（规划产出任务 → 日程安排时间？）
- 用一句话说清楚：规划模块的核心价值是什么？

### 2. 目标体系设计
- 目标层级如何设计？（大目标 → 阶段目标 → 每日任务？还是更简洁的两层？）
- 支持哪些目标类型？
  - 学业目标（考研、GPA、英语四六级）
  - 技能目标（编程、设计、语言）
  - 习惯目标（健身、早起、阅读）
  - 自定义目标
- 时间维度：日/周/月/学期/年度，MVP应该先支持哪几个？

### 3. 游戏化机制设计（核心差异化）
- "星空领域"视觉激励如何设计？（完成任务解锁星星？积累点亮星座？）
- 经验值/等级体系：是否有成长感？如何避免无聊？
- 连续打卡机制：断签处理怎么设计不让用户放弃？（补签卡？宽限期？）
- 成就/徽章系统：有哪些有意思的成就设计？（如"7天早起达人"、"期末生存者"）
- 与校园墙联动：完成里程碑后是否自动生成可分享的"成就卡片"？

### 4. AI规划助手
- AI如何帮助用户制定计划？（用户说"我要考研"，AI给出多少天、每天几小时的拆解？）
- AI能否基于用户的历史完成率动态调整计划难度？
- 与智能日程的AI识别导入有何不同？（规划AI更关注长期目标拆解）
- 用户最常见的规划场景是什么？给出3个预设模板建议

### 5. 社交激励机制
- "好友互相监督"如何实现？（查看好友今日完成情况？互相点赞加油？）
- 是否需要"公开打卡"功能？（发布到校园墙？）
- 班级/社团的"集体目标"是否有价值？（全班一起备考、社团活动倒计时）
- 社交激励会不会带来压力反而让用户不想用？如何平衡？

### 6. 数据统计与回顾
- 每周/月自动生成"成长报告"？内容包含什么？
- 年度总结（类似微信年度报告）是否有传播价值？
- 哪些数据可视化对大学生最有意义？（完成率趋势、专注时长、目标进度）

### 7. MVP范围建议
- 如果只开发3个核心功能，你推荐哪3个？
- 哪些游戏化元素可以后期再加，MVP不必做？
- 这个模块的上线时机：是独立上线，还是应该等日程模块稳定后再推？

## 输出格式

```
## 与日程模块的边界定义
## 目标体系设计方案

```
