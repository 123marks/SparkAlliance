# 星火规划模块 —— 用户体验与视觉交互设计方案

> 基于暗黑玻璃拟态风格的深度设计思考
> 设计目标：让目标管理从「负担」变成「期待」

---

## 一、首页设计方案（Daily Dashboard）

### 1.1 核心设计原则：「3秒定律」

用户打开规划模块的3秒内必须回答一个问题：**"我今天该做什么？"**

### 1.3 今日任务 vs 目标进度 —— 优先级判定

**结论：今日任务占视觉重心60%，目标进度占40%**

| 维度 | 今日任务 | 目标进度 |
|------|----------|----------|
| **心理距离** | 近（24小时内） | 远（数周/数月） |
| **行动压力** | 高（不做就断连） | 低（还有时间） |
| **视觉位置** | 上半区，首屏必见 | 下半区，需滑动 |
| **交互权重** | 大按钮，强引导 | 卡片点击，弱引导 |
| **数据展示** | 待办清单形式 | 进度条+百分比 |

**设计理由**：行为心理学中的"执行意图"理论——明确的"如果-那么"计划（If-Then Planning）能显著提升目标达成率。今日任务就是用户的"那么"。

- 提供3个热门模板作为低摩擦入口
- 按钮使用脉冲动画吸引点击

### 1.5 时间感知设计 —— 早晨 vs 晚上的差异化体验

**早晨模式（6:00-11:59）**：
- 问候语："早安，今天的征途即将开始 🌅"
- 视觉色调：暖金色渐变 overlay
- 默认展示：今日任务清单（待办导向）
- 特殊提示：如有昨日未完成任务，显示"昨日延续"标签

**下午模式（12:00-17:59）**：
- 问候语："下午好，保持节奏 ⚡"
- 视觉色调：保持品牌蓝紫色调
- 默认展示：今日任务 + 完成度统计

**晚上模式（18:00-23:59）**：
- 问候语："今晚的成果，明日的基石 🌙"
- 视觉色调：深紫+靛蓝色调，星星更亮
- 默认展示：今日完成总结 + 明日任务预览
- 特殊功能：一键生成"今日打卡分享卡片"

**深夜模式（00:00-5:59）**：
- 问候语："星辰不负赶路人 ✨"
- 弱打扰设计：静音提醒，柔和动画
- 功能限制：仅允许打卡，不建议创建新目标

### 2.4 截止日期设计 —— 紧迫感 vs 焦虑感的平衡

**核心洞察**：倒计时要让人感受到"时间珍贵"而不是"时间不够"

```
│                                                             │

```

**设计要点**：
1. **正向倒计时**：显示"还有X天"而非"只剩X天"，语言框架效应
2. **进度可视化**：环形进度条 + 百分比 + 已坚持天数

**颜色策略**：
- 时间充裕（>50%）：蓝绿色系，平静
- 时间适中（20-50%）：黄色系，提醒
- 时间紧迫（<20%）：橙色系，警告但不使用红色（避免恐慌）

### 3.2 打卡动效设计 —— 游戏化的成就感反馈

**核心原则**：每一次打卡都应该是一次"微型庆祝"

**动效方案组合**：

**Layer 1 - 即时反馈（0-300ms）**：
```css
/* 按钮点击反馈 */
.check-button:active {
  transform: scale(0.95);
  transition: transform 0.1s ease;
}

/* 完成瞬间 */
.check-button.completed {
  transform: scale(1.1);
  background: linear-gradient(135deg, #f59e0b 0%, #ef4444 100%);
  box-shadow: 0 0 30px rgba(245, 158, 11, 0.6);
}
```

**Layer 2 - 粒子爆发（300ms-800ms）**：
- 从按钮中心发射12-16颗"星尘粒子"
- 粒子颜色：金黄 → 橙红渐变
- 粒子运动：向外扩散 + 重力下落 + 渐隐
- 音效（可选）：清脆的"叮"声

**Layer 3 - 全局反馈（800ms-1500ms）**：
- 任务卡片变为金色边框 + 完成印章（✓）
- 连续打卡数字 +1 的翻转动画
- 进度条向前推进的流动动画

**代码级动效示意**：
```typescript
// 打卡成功动画序列
async function playCheckInAnimation() {
  // 1. 按钮反馈
  await animateButtonPress();
  
  // 2. 粒子爆发（使用Canvas或Lottie）
  particleSystem.explode({
    origin: buttonPosition,
    count: 16,
    colors: ['#f59e0b', '#f97316', '#ef4444'],
    spread: 100,
    duration: 500
  });
  
  // 3. 卡片状态变更
  await Promise.all([
    animateCardCompletion(),
    animateStreakIncrement(),
    animateProgressAdvance()
  ]);
  
  // 4. 如果是今日全部完成，触发特殊庆祝
  if (allTasksCompleted) {
    triggerDailyCompleteCelebration();
  }
}
```

## 四、星空领域视觉方案（核心差异化设计）

### 4.1 方案评估与最终选择

| 方案 | 优势 | 劣势 | 契合度 |
|------|------|------|--------|
| **A. 个人星图** | 目标关联感强，完成度=亮度，有归属感 | 技术实现复杂，多目标时拥挤 | ⭐⭐⭐⭐ |
| **B. 星云成长** | 视觉冲击力强，梦幻感 | 抽象度过高，难以感知具体进展 | ⭐⭐⭐ |
| **C. 像素星空** | 复古有趣，可形成图案 | 与整体玻璃拟态风格冲突 | ⭐⭐ |

**最终选择：方案A - 个人星图（Personal Constellation）**

**核心理由**：
1. **隐喻完美契合**：目标 = 星星，完成 = 点亮，与平台"星火"名称呼应
2. **情感共鸣**：每个人都需要一个"属于自己的星空"
3. **可扩展性**：支持多目标、长期积累、社交展示
4. **技术可行**：可基于现有ParticleBackground组件扩展

### 4.2 个人星图设计方案

**核心概念**：
- 每创建一个目标，就在星空中生成一颗"目标星"
- 目标的完成进度决定星星的亮度（0% = 暗淡，100% = 璀璨）
- 目标达成后，星星变为"永恒星"，永久留在星图中
- 多个目标组成"个人星座"，连线形成独特图案

**视觉细节**：

```
┌─────────────────────────────────────────────────────────────┐
│  我的星空领域                                    [分享] [?] │
```

4. **缩放探索**：双指缩放可查看更大范围的星空（历史目标）

### 4.3 星空独特性设计

**如何让每个用户的星空独一无二？**

**维度1：星座连线算法**
- 自动连线：根据目标创建时间和类型自动生成星座图案
- 手动连线：用户可自定义星星之间的连接关系
- 意义生成：AI根据连线形状给出星座名称（如"考研猎户座"）

**维度2：星星分布算法**
- 基于目标创建时间的伪随机分布（可复现）
- 避免重叠的力导向布局
- 活跃目标在视觉中心，历史目标在外围

**维度3：环境变化**
- 真实时间同步：白天星空暗淡，夜晚星空璀璨
- 季节变化：不同季节背景星座不同
- 特殊节日：新年流星雨、中秋满月等彩蛋

### 4.4 展示形式选择

## 七、失败状态的友好处理

### 7.1 连续未打卡检测与处理

**分层处理策略**：

| 未打卡天数 | 系统行为 | 文案策略 |
|------------|----------|----------|
| **1天** | 不打扰 | 无提醒（给用户容错空间） |
| **2天** | 首页弱提示 | "昨天缺席了，今天补回来？" |
| **3天** | 推送提醒 | "3天没见了，星星们想你了 ✨" |
| **7天** | 目标暂停询问 | "这个目标暂时搁置了吗？" |
| **14天** | 目标自动暂停 | 保留数据，可一键重启 |

**3天未打卡提醒设计**：

```
┌─────────────────────────────────────────────────────────────┐

│                                                             │

```

### 7.2 "我失败了"的体面退出

**核心原则**：不评判、不羞辱、保留尊严

**放弃流程设计**：

```
Step 1: 触发放弃意图
用户点击"放弃目标" → 不立即执行，先询问原因

Step 2: 了解原因（数据收集 + 情感疏导）
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  是什么原因让你想放弃这个目标？                              │
│  （这不是失败，只是帮助我们更好地设计产品）                   │
│                                                             │
│  ○ 目标太难，无法坚持                                        │
│  ○ 生活发生变化，没时间了                                    │
│  ○ 找到了更好的方式                                          │
│  ○ 只是需要暂停一下                                          │
│  ○ 其他原因                                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘

Step 3: 根据原因给出不同响应

- 选择"目标太难"：
  "要不要试试把目标拆得更小一些？
   比如从'每天背100个单词'变成'每天背20个单词'？"
  按钮：[调整目标] [仍然放弃]

- 选择"只是需要暂停"：
  "好的，我们会保留你所有的进度数据
   当你准备好了，随时可以回来继续"
  按钮：[暂停目标] [取消]

- 选择"其他原因"：
  "无论如何，你已经坚持了 X 天
   这段经历不会消失，它会成为你星空的一部分"
  按钮：[确认放弃]
```

**放弃后的处理**：
- 目标标记为"已暂停"而非"已失败"
- 对应的星星变为"休眠状态"（暗淡但可见）
- 保留所有历史数据
- 首页显示"休眠目标"列表，可随时一键重启
- 重启时恢复所有进度

### 7.3 补签功能设计

**是否支持补签**：是，但有限制

**补签规则**：
1. **时间限制**：仅支持补签过去7天内的记录
2. **次数限制**：每月最多补签3次
3. **积分惩罚**：补签需要消耗"星火积分"，正常打卡获得积分
4. **数据标记**：补签的任务用特殊标识（如星号*），与正常打卡区分

**补签流程**：

```
┌─────────────────────────────────────────────────────────────┐
│  补签 3月25日                                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  选择要补签的任务：                                          │
│                                                             │
│  ☑️ 背单词 50 个                                            │
│  ⭕ 高数复习（已无法补签，超过7天）                          │
│                                                             │
│  补签原因（可选）：                                           │
│  [____________]                                             │
│                                                             │
│  本次补签将消耗 10 积分（当前余额：45 积分）                 │
│                                                             │
│  [确认补签]                                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**防滥用设计**：
- 补签记录不参与"连续打卡天数"统计
- 补签次数过多时，系统提示"补签是为了应对特殊情况，不要形成依赖哦"
- 补签的任务在热力图上用不同颜色/透明度标识

## 九、设计总结

### 核心设计理念

> **"让目标管理从负担变成期待"**

通过以下设计策略实现：

1. **游戏化但不儿戏**：用星星、星座等隐喻激发探索欲，但保持目标管理的严肃性
2. **正向激励而非负向惩罚**：没有"失败"，只有"暂停"；没有"落后"，只有"节奏不同"
3. **仪式感但不繁琐**：打卡有庆祝，但不强制写心得；有社交，但不制造焦虑
4. **个性化但不复杂**：每个人的星空独一无二，但不需要复杂的设置

### 差异化亮点

| 传统目标管理App | 星火规划 |
|-----------------|----------|
| 冷冰冰的进度条 | 有生命的星空 |
| 催命般的提醒 | 温柔的陪伴 |
| 失败的羞耻感 | 暂停的从容 |
| 孤独的坚持 | 朋友的鼓励 |
| 机械的打卡 | 游戏化的成就 |

### 期待的效果

- **日活指标**：用户每天打开不是为了"完成任务"，而是为了"看看自己的星空"
- **留存指标**：放弃率降低，因为"放弃"被重新定义为"暂停"
- **口碑指标**：用户愿意分享的不是"我用了个打卡App"，而是"你看我的星空"
