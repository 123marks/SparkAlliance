# iflow — 星火规划模块头脑风暴指令

## 任务背景

星火校园（Spark Alliance）是面向大学生的综合平台，整体风格为暗黑系 + 玻璃拟态 + 星火配色。

**新模块：星火规划（目标与任务）**
游戏化成长模块，核心体验是"定目标 → 拆任务 → 每日打卡 → 看成长"。
目标用户痛点：大学生有宏大目标（考研/健身/学技能）但缺乏坚持机制，容易三分钟热度。

参考现有风格：`frontend/src/components/GlassCard.vue`、`frontend/src/components/ParticleBackground.vue`

---

## 你的任务

从**用户体验流程与视觉交互设计**角度进行头脑风暴：

### 1. 首页设计（用户每天打开的界面）
- 规划首页应该展示什么？让用户一眼看到"今天该做什么"
- 今日任务列表 vs 整体目标进度，哪个更重要、更靠前？
- 空状态设计（还没创建任何目标时）如何激励用户开始？
- 早晨进来和晚上进来的展示内容应该一样吗？（时间感知设计）

### 2. 创建目标体验
- 创建目标的最短路径：几步最合适？（用户说"我要考研" → AI拆解 → 确认 → 创建完成）
- AI辅助创建：用户输入一句话，AI生成目标详情和任务拆解，交互如何设计？
  （弹窗？全屏引导流程？右侧预览面板？）
- 自定义 vs 模板：是否提供"考研100天"、"健身打卡"等预设模板？
- 截止日期设置：倒计时设计如何让用户感受到紧迫感而不是焦虑？

### 3. 每日打卡体验（最高频交互）
- 打卡入口在哪？（首页固定区域？底部导航角标提醒？）
- 打卡动效：完成一个任务的反馈动画如何设计有成就感？（爆炸粒子？星星飞出？）
- 打卡时是否需要填写备注/心情？（增加仪式感 vs 增加门槛）
- 今日全部完成时的庆祝界面如何设计？

### 4. 星空领域可视化
- "星空领域"是整个规划模块的视觉核心，如何设计？
  方案A：个人星图——每个目标是一颗星，完成度决定亮度，组成个人专属星座
  方案B：星云成长——积分越多星云越大越亮，有梦幻感
  方案C：像素星空——完成任务点亮像素格，俯瞰时形成图案（像Pokemon地图）
  你倾向哪种？为什么？
- 这个视觉元素应该是全屏沉浸式体验，还是嵌入在首页的一个区域？
- 如何让不同用户的星空看起来独一无二？

### 5. 社交激励展示
- 好友的打卡动态如何展示？（类似运动App的好友排行榜？Feed流？）
- "互相监督"的提醒文案怎么写不显得催促？
- 完成目标里程碑后自动生成分享卡片的设计：包含什么内容？用什么视觉风格？

### 6. 进度与数据可视化
- 目标进度条的设计：线性进度条 vs 环形进度 vs 里程碑节点
- 习惯打卡记录的热力图（类似GitHub）：多大尺寸合适？颜色方案？
- 周/月成长报告的可视化风格：简洁数字卡片 vs 炫酷图表 vs 故事式叙述？

### 7. 放弃/失败状态的体验
- 用户很久没打卡时（3天/7天），系统如何处理？（温柔提醒？目标暂停？自动放弃？）
- "我失败了"的界面如何设计不让用户感到羞耻？（数据保留？重启按钮？）
- 补签功能：是否支持？如何避免被滥用影响数据真实性？
