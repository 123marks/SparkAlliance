# Qoder CLI — 星火卡罗牌功能头脑风暴指令

## 任务背景

星火校园（Spark Alliance）是面向大学生及青年群体的综合平台，技术栈为 Vue 3 + TypeScript + Vite + Supabase，整体风格为暗黑系 + 玻璃拟态 + 星火配色。

**新功能：星火卡罗牌**
一个融合塔罗牌玄学文化与AI解读的趣味功能模块，面向大学生群体，提供每日占卜、问题解读、星座运势等轻量娱乐体验。

参考项目文档：`docs/project/SparkAlliance.md`（了解项目定位与调性）

---

## 你的任务

从**产品定位与功能架构**角度进行头脑风暴，回答以下问题：

### 1. 功能定位与边界
- 这个功能叫"卡罗牌"而不是"塔罗牌"，星火校园如何做出自己的差异化？
  （纯娱乐？结合AI深度解读？结合日程/心情？结合校园场景？）
- 核心用户场景是什么？（考试前测运势、感情困惑、每日心理暗示、还是纯好玩？）
- 功能边界：是独立模块，还是嵌入到 AI 助手 / 首页 / 个人中心的轻量入口？

### 2. 卡牌体系设计
- 星火卡罗牌是沿用传统78张塔罗体系，还是自创更贴合校园场景的卡牌？
  例：「开学季」「期末周」「社团招新」「暗恋」等校园专属主题牌
- 正位/逆位设计是否保留？对大学生来说理解成本如何？
- 建议设计多少张牌？MVP阶段最少几张可以跑通体验？

### 3. AI 解读机制
- AI 解读内容如何个性化？（结合用户提出的问题、当前情绪、星座、时间？）
- 解读风格如何区分？（严肃解析风 / 轻松调侃风 / 诗意文学风）
- 大学生最可能输入什么问题？预设问题模板应该有哪些？

### 4. 社交传播设计
- 抽到的牌如何一键分享到校园墙？
- 是否设计"每日同款"机制（全校当天抽同一张牌，对比解读）？
- 好友之间互相"抽牌占卜"是否有价值？

### 5. 数据与个性化
- 历史抽牌记录有价值吗？如何呈现？（趋势？词云？）
- 是否需要用户档案（星座、MBTI、生日）来增强解读准确性？
- 抽牌频率限制：每天1次免费，多次需要什么？（积分？会员？）

### 6. 与现有模块联动
- 抽到"考试运势卡"时，能否一键跳转日程模块添加复习提醒？
- 抽到"社交运势卡"时，能否联动校园墙发帖？
- 个人中心是否展示"你的守护星火牌"？

## 输出格式

```
## 功能定位建议
## 卡牌体系设计方案
## AI解读机制设计
## 社交传播玩法
## MVP最小可行方案（张数/功能/数据表）
## 与现有模块联动规划
## 综合评分与优先级建议
```
