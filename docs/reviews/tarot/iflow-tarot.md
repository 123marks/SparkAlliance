# iflow — 星火卡罗牌功能头脑风暴指令

## 任务背景

星火校园（Spark Alliance）是面向大学生的综合平台，整体风格为暗黑系 + 玻璃拟态 + 星火配色。

**新功能：星火卡罗牌**
融合塔罗文化与AI解读的趣味占卜模块，核心体验是"抽牌 → 提问 → AI解读 → 分享"。

参考现有风格：`frontend/src/pages/app/CampusWall.vue`（暗黑玻璃拟态参考）

---

## 你的任务

从**用户体验流程与视觉交互设计**角度进行头脑风暴：

### 1. 第一次体验设计（关键时刻）
- 用户第一次进入卡罗牌页面，看到什么？如何在3秒内让用户"想要抽一张"？
- 空牌桌状态（还没抽牌时）如何设计才有神秘感？
- 新手引导需要吗？如果需要，几步最合适？

### 2. 核心抽牌流程
```
理想的最短路径是什么？
进入页面 → ? → ? → 看到解读结果
```
- 用户必须输入问题才能抽牌，还是可以"随缘一抽"？
- 抽牌前的"仪式感"如何设计？（深呼吸倒计时？闭眼动效？摇一摇？）
- 抽牌翻转动画的节奏：多久翻完最有惊喜感？（0.3s？0.8s？分段式？）

### 3. 解读结果页面设计
- AI解读文字如何呈现才有"神谕感"？（逐字打字机效果？淡入？从中心扩散？）
- 解读内容分区：牌义说明 vs AI个性解读，如何区分展示不乱？
- 结果页的情绪设计：不同类型的牌（吉牌/凶牌/中性牌）颜色氛围如何区分？

### 4. 牌面视觉风格
- 星火卡罗牌的视觉风格建议：
  选项A：赛博朋克+星空（与项目暗黑风统一）
  选项B：中国水墨+星火元素（差异化，有文化感）
  选项C：极简几何线条（现代感，制作成本低）
  你倾向哪种？为什么？
- MVP阶段牌面图片从哪里获取最可行？（AI生成？开源素材？SVG自绘？）

### 5. 分享机制设计
- 分享到校园墙时，分享卡片如何设计？（牌面图+解读摘要？还是神秘感十足不透露内容？）
- "今日运势" 分享图尺寸和内容建议（参考微博/小红书分享卡片）
- 是否设计"@好友帮你解读"的社交玩法？

### 6. 频率与会员体验
- 每日1次免费的限制，用户遇到时的情绪如何？提示文案怎么写不让人反感？
- 付费解锁更多次数的入口设计：弹窗？底部栏？还是融入在结果页？
- "明日运势预告"（提前告知明天的牌）是否是一个好的付费吸引点？

### 7. 历史记录体验
- 历史抽牌记录页面如何设计？（时间轴？卡片瀑布流？月历视图？）
- 是否显示"你最常抽到的牌"？用户会喜欢这个统计吗？
- 与日程模块联动：在某个重要日期（考试/面试）前抽的牌，结合事件显示？
