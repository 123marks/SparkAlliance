# iflow — 智能日程模块头脑风暴指令

## 任务背景

星火校园（Spark Alliance）是面向大学生的综合平台，技术栈为 Vue 3 + TypeScript + Vite + Supabase。
设计文档：`docs/MODULE-SCHEDULE.md`（请完整阅读）
项目风格参考：`frontend/src/pages/app/CampusWall.vue`（暗黑系 + 玻璃拟态 + 星火配色）

## 你的任务

以 iflow 的视角，从**用户体验流程**和**产品交互设计**角度进行头脑风暴：

### 1. 用户旅程分析
- 大学生首次使用智能日程的完整流程是什么？（从进入页面到第一个事件创建完成）
- 哪个环节最容易流失用户？如何降低认知成本？
- 空状态（没有任何事件时）如何设计才能引导用户快速上手？

### 2. 核心交互流程
- 点击日历空白格子 → 快速新建事件（Quick Add）：比打开完整弹窗更高效，如何实现？
- 事件详情查看 → 编辑 → 保存，这个流程的最短路径是什么？
- 删除事件时，重复事件应该弹出"删除本条"还是"删除所有"的选择吗？

### 3. 视图切换体验
- 月/周/日三种视图之间切换时，如何保持"日期焦点"不丢失？
  （例如：在月视图点击3月18日 → 切换到日视图，应该直接跳到3月18日）
- 从日视图"向左滑"切换到前一天，是否需要手势支持？（移动端）
- 今天按钮点击后的动画效果如何设计？

### 4. 大学生特有的使用场景
- **课表导入引导**：MVP阶段无法自动导入，如何设计一个"手动录入课表"的引导流程，让用户不觉得麻烦？
- **考试临近提醒**：当检测到7天内有考试事件时，首页/日程首屏应该如何突出显示？
- **假期/节假日**：法定节假日如何在日历上优雅标注，不影响正常事件显示？

### 5. 与校园墙的协同体验
- 如果用户在校园墙发帖时想关联一个日程事件（"今晚图书馆自习，欢迎一起"），入口在哪里？交互流程是什么？
- 好友在校园墙发布了带日程的帖子，我想"一键加入我的日程"，这个功能怎么设计？

### 6. 通知与提醒体验
- Web Notification 权限请求的时机：进入日程页就请求 vs 用户第一次设置提醒时请求？哪个转化率更高？
- 提醒弹出时的视觉样式，如何与整体暗黑风格保持一致？
- "稍后提醒"（Snooze）功能是否必要？大学生场景下多久合适（5分钟/15分钟）？

### 7. 错误与边界情况
- 网络断线时，用户新建的事件如何处理（乐观更新 + 离线队列）？
- 时间冲突检测到后，UI如何呈现警告而不打断用户操作流程？
- 跨午夜的事件（23:00 - 01:00）在日视图中如何显示？
