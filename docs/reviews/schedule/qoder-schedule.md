# Qoder CLI — 智能日程模块头脑风暴指令

## 任务背景

星火校园（Spark Alliance）是面向大学生的综合平台，技术栈为 Vue 3 + TypeScript + Vite + Supabase。
设计文档位于：`docs/MODULE-SCHEDULE.md`（请完整阅读后再作答）。
路由已就绪：`/app/schedule` → 当前指向 `Placeholder.vue`，开发完成后替换为 `Schedule.vue`。

## 你的任务

深度阅读 `docs/MODULE-SCHEDULE.md`，结合以下维度进行头脑风暴，输出你的完整建议：

### 1. 架构层面
- MVP 5个组件是否够用？有无遗漏的核心交互场景？
- `useSchedule.ts` composable 与 `scheduleStore.ts` 职责如何划分最合理？
- Supabase 实时订阅（Realtime）在日程模块是否有价值？（例如多端同步）

### 2. 数据层面
- `schedule_events` 表中 `recurrence_days INT[]` 存储重复星期几，是否有更优方案？
- 重复事件在前端展开 vs 后端预生成，各有什么利弊？大学生场景下哪个更适合？
- `event_reminders` 表是否必要？能否直接在前端用 localStorage 持久化提醒队列？

### 3. 交互层面
- 周视图中，如果同一时间段有多个事件重叠，如何优雅处理布局？
- 新建事件弹窗中，日期时间选择用原生 `<input type="datetime-local">` 还是自定义组件？
- 拖拽调整事件时间（V2功能），在移动端如何兼容触摸事件？

### 4. 校园场景特有问题
- 课表导入：大多数高校教务系统无开放API，只能靠模拟登录或手动录入。MVP阶段最务实的方案是什么？
- 考试周、法定节假日数据从哪里获取？有无免费可用的API？
- "第N周"的计算依赖学期开始日期，如何让用户方便地配置？

### 5. 性能与体验
- 月视图渲染大量事件时，Vue 的虚拟滚动是否必要？（通常一个月不超过100个事件）
- 当前时间红线在周/日视图中如何精确定位？用 CSS `top` 百分比还是绝对像素？
- 骨架屏加载状态如何设计，与校园墙保持视觉一致？

### 6. 与现有模块联动
- Profile 页面展示"本周课程摘要"，数据接口如何设计？
- 校园墙发帖关联日程（"我今晚在图书馆自习，欢迎加入"），需要哪些数据结构支持？
