## 一、项目核心定位

### 1.1 项目概述

**项目名称**: 星火校园 (Spark Alliance)  
**核心定位**: 以AI为特色入口，深度融合社交、生活服务与个人管理的校园超级App  
**目标用户**: 大学生 → 青年群体（学生、职场人士、自由职业者等）  
**发展路径**: 校园版 → 社区版 → 国外版

### 1.2 商业模式

- **前期**: 免费AI和工具功能吸引用户（引流）
- **中期**: 二手交易撮合费和增值服务（初步盈利）
- **后期**: B端商家、会员体系、高利润商品（规模化盈利）

## 二、七大功能模块

### 功能一：AI模式（流量入口与特色）

**核心功能：**
- 集成多模型（DeepSeek、豆包等）
- 本地授权方案（不使用免费API）
- 悬浮窗设计
- 历史记录侧边栏

**技术方案：**
- ⚠️ **重要决策**: 采用本地授权方案
- 用户在本地登录DeepSeek/豆包等模型
- App检测本地登录状态并获取授权
- App内独立对话记录，不同步其他平台

**注意事项：**
- 需要研究各平台的授权机制
- 确保数据隔离和安全性
- 用户必须明确授权

## 四、技术选型

### 4.1 数据库方案

**决策：**
- **前期**: 使用Supabase
- **后期**: 可迁移到其他数据库

**原因：**
- Supabase提供完整后端服务
- 快速开发，降低初期成本
- PostgreSQL兼容性好

**注意事项：**
- ⚠️ 使用标准SQL，避免Supabase特有功能
- ⚠️ 预留数据导出接口
- ⚠️ 设计数据迁移方案

---

### 4.2 前端框架

**选项：**
- React Native
- Flutter

**待确定：**
- 根据团队技术栈决定
- 考虑性能需求

---

### 4.3 后端技术

**推荐：**
- Node.js + Express / NestJS
- 或 Go

---

### 4.4 AI模型接入

**方案：本地授权方案**

**流程：**
1. 检测用户本地是否登录DeepSeek/豆包等
2. 请求用户授权App使用该模型
3. 获取授权Token
4. 在App内调用模型API
5. 独立存储对话记录

**优势：**
- 不需要免费API的复杂接入
- 用户体验更好
- 数据更安全

**挑战：**
- 需要各平台支持授权
- 技术实现需要研究
