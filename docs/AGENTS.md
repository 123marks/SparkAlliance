## 🔧 技术决策记录

### 决策1: 数据库选择
**时间**: 2026-03-15  
**决策**: 前期使用Supabase，后期可迁移  
**原因**: 
- Supabase提供完整的后端服务
- 快速开发，降低初期成本
- PostgreSQL兼容性好，便于迁移

**风险**: 
- 迁移成本需要提前规划
- 需要设计数据迁移方案

**应对**: 
- 使用标准SQL，避免Supabase特有功能
- 预留数据导出接口

---

### 决策2: AI模型接入方式
**时间**: 2026-03-15  
**决策**: 采用本地授权方案，不使用免费API  
**原因**: 
- 免费API接入复杂，限制多
- 本地授权更安全，用户体验更好
- 用户已有账号，无需重复注册

**风险**: 
- 技术实现难度较高
- 需要各平台支持授权

**应对**: 
- 先研究DeepSeek授权机制
- 如果不支持，考虑其他方案

---

### 决策3: 前端框架选择
**时间**: 待定  
**决策**: React Native 或 Flutter  
**原因**: 
- 一套代码开发iOS和Android
- 社区成熟，生态完善

**待确认**: 
- 团队技术栈
- 性能需求
