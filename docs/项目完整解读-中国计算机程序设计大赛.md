# 一、项目概述

## 1.1 项目基本信息

**项目名称**：星火联盟（Spark Alliance）  
**项目类型**：校园综合服务平台  
**核心特色**：AI驱动的校园超级App  
**目标用户**：大学生 → 青年群体（学生、职场人士、自由职业者）  
**发展路径**：校园版 → 社区版 → 国际版  

## 1.2 项目背景

### 1.2.1 市场痛点分析

**当前校园服务存在的问题：**

1. **信息孤岛**
   - 校园服务分散在不同平台（教务系统、社交软件、二手群等）
   - 学生需要在多个App间切换，效率低下
   - 信息获取成本高，错过重要信息

2. **信任缺失**
   - 二手交易缺乏信任机制，欺诈风险高
   - 校园社交缺乏身份认证，真实性难保证
   - 学习资源质量参差不齐，难以筛选

3. **效率低下**
   - 选课缺乏数据支持，盲目选择
   - 学习问题无法及时得到解答
   - 日程管理工具单一，缺乏智能提醒

4. **社交局限**
   - 校园社交圈子小，难以拓展人脉
   - 缺乏与学长学姐的有效沟通渠道
   - 学习交流平台匮乏

### 1.2.2 市场机会

**校园服务市场规模：**
- 中国在校大学生：约4,000万人
- 校园服务市场规模：约1,000亿元/年
- 二手交易市场：约200亿元/年
- 在线教育市场：约3,000亿元/年

**用户需求趋势：**
- AI工具需求爆发式增长
- 校园数字化服务需求增加
- 二手交易需求持续增长
- 社交+学习融合需求明显

## 1.3 项目愿景

**短期愿景（1年内）：**
- 成为所在高校最受欢迎的校园服务平台
- 服务10,000+学生用户
- 建立完善的校园服务生态

**中期愿景（3年内）：**
- 覆盖全国100+高校
- 服务100万+学生用户
- 成为校园服务领域头部平台

**长期愿景（5年内）：**
- 拓展至青年群体（职场人士、自由职业者）
- 国际化发展，服务海外留学生
- 成为青年群体必备的综合服务平台

# 二、核心定位与价值主张

## 2.1 核心定位

**一句话定位**：  
以AI为特色入口，深度融合社交、生活服务与个人管理的校园超级App

**定位拆解：**

1. **AI为特色入口**
   - AI助手作为核心功能，吸引用户
   - 多模型集成，提供专业AI服务
   - AI赋能各个功能模块，提升体验

2. **社交为核心粘性**
   - 校园社交建立用户关系链
   - 社交互动提高用户留存
   - 社交推荐增强内容分发

3. **生活服务为变现路径**

   - 增值服务提供持续收入

4. **个人管理为工具属性**
   - 日程管理提高用户依赖
   - 目标规划增强用户粘性

# 三、功能模块详解

## 3.1 功能模块总览

星火联盟包含**16个核心功能模块**，覆盖学习、生活、社交、管理四大场景：

```
功能模块架构
├── 学习场景（5个模块）
│   ├── AI助手 - 智能答疑与学习辅导
│   ├── 智能日程 - 时间管理与提醒
│   ├── 学习资源 - 资源共享与获取
│   ├── 星火自习室 - 在线学习空间
│   └── 星火规划 - 目标管理与成长
│
├── 生活场景（4个模块）
│   ├── 校园购物 - 二手交易与生活服务
│   ├── 健康打卡 - 健康管理与习惯养成
│   ├── 星火资讯 - 校园信息聚合
│   └── 星火塔罗 - 娱乐与心理疏导
│
├── 社交场景（4个模块）
│   ├── 校园墙 - 校园社区与动态分享
│   ├── 星火伴侣 - 社交与好友管理
│   ├── 星火消息 - 即时通讯
│   └── 星火人才 - 人才展示与匹配
│
└── 管理场景（3个模块）
    ├── 个人中心 - 个人信息与设置
    ├── 学长推荐 - 经验分享与咨询
    └── 星火共创 - 协作与项目管理
```

## 3.2 核心功能详解

### 3.2.1 AI助手（核心特色功能）

**功能定位**：  
AI助手是星火联盟的核心特色和流量入口，通过集成多模型提供智能答疑、学习辅导、生活建议等服务。

**核心功能：**

1. **多模型集成**
   - 支持DeepSeek、豆包、GPT等多种AI模型
   - 用户可选择不同模型或使用平台默认模型
   - 支持用户自填API Key，降低平台成本

2. **智能答疑**
   - 学科问题解答（数学、英语、编程等）
   - 分步骤详细讲解
   - 举一反三推荐相关知识点

3. **学习辅导**
   - 作业辅导与解题思路
   - 论文润色与修改建议
   - 代码审查与优化建议

4. **生活建议**
   - 选课建议与数据分析
   - 职业规划与方向指导
   - 时间管理与效率提升

**技术实现：**
- 前端：Vue 3 + TypeScript
- 后端：Supabase Edge Functions
- AI集成：多模型API代理
- 数据存储：对话历史记录

**创新点：**
- 混合模式：平台额度 + 用户自持API Key
- 上下文记忆：独立对话记录，不同步其他平台
- 个性化：根据用户画像调整回复风格

### 3.2.2 校园墙（核心社交功能）

**功能定位**：  
校园墙是星火联盟的核心社交功能，类似朋友圈的校园社区，支持动态发布、互动评论、话题讨论。

**核心功能：**

1. **动态发布**
   - 文字、图片、视频发布
   - 话题标签与分类
   - 匿名发布（表白墙等）

2. **互动功能**
   - 点赞、评论、分享
   - 关注作者、收藏动态
   - 举报违规内容

3. **内容分类**
   - 动态分享：日常分享
   - 表白墙：匿名表白
   - 求助问答：问题求助
   - 二手交易：闲置物品
   - 失物招领：丢失物品

4. **智能推荐**
   - 基于兴趣标签推荐
   - 基于社交关系推荐
   - 基于热度排序推荐

**技术实现：**
- 实时更新：WebSocket推送
- 图片存储：Supabase Storage
- 内容审核：AI自动审核 + 人工审核
- 数据库：PostgreSQL

**创新点：**
- 多分类体系：满足不同场景需求
- 匿名机制：保护隐私，鼓励表达
- 智能推荐：提高内容分发效率

### 3.2.3 校园购物（核心变现功能）

**功能定位**：  
校园购物是星火联盟的核心变现功能，基于校园场景的C2C二手交易平台，支持校内面交和配送。

**核心功能：**

1. **商品发布**
   - 商品信息填写（标题、描述、价格、图片）
   - 商品分类（教材、电子产品、生活用品等）
   - 成色标注与价格建议

2. **交易流程**
   - 浏览与搜索商品
   - 即时通讯沟通
   - 约定交易时间地点
   - 确认交易完成

3. **信任机制**
   - 学生认证体系
   - 交易评价系统
   - 信用评分展示
   - 举报与风控

4. **交易保障**
   - 平台担保交易（可选）
   - 纠纷处理机制
   - 售后服务支持

**技术实现：**
- 即时通讯：WebSocket
- 图片上传：Supabase Storage
- 支付集成：微信支付/支付宝
- 地理位置：LBS定位

**创新点：**
- 校内面交：降低物流成本
- 学生认证：建立信任基础
- 信用体系：促进诚信交易

### 3.2.4 智能日程（个人管理功能）

**功能定位**：  
智能日程是星火联盟的个人管理功能，提供可视化日历、事件提醒、课表导入等服务。

**核心功能：**

1. **日程管理**
   - 日历视图（日/周/月）
   - 事件创建与编辑
   - 颜色标签分类
   - 重复事件设置

2. **智能提醒**
   - 多时间点提醒
   - 提醒方式选择（应用内/邮件/短信）
   - 智能提醒时间建议

3. **课表导入**
   - 教务系统课表导入
   - 手动添加课程
   - 课程冲突检测

4. **AI增强**
   - AI日程规划建议
   - 智能时间分配
   - 效率分析与优化

**技术实现：**
- 日历组件：自定义Vue组件
- 数据存储：PostgreSQL
- 提醒服务：定时任务 + 推送通知
- AI集成：日程规划AI

**创新点：**
- AI规划：智能时间管理建议
- 多视图：满足不同使用习惯
- 课表集成：学生场景优化

## 4.2 前端技术栈详解

### 4.2.1 核心技术

**Vue 3 + TypeScript**
- **选择理由**：
  - Vue 3性能优秀，Composition API灵活
  - TypeScript提供类型安全，减少错误
  - 生态成熟，社区活跃

- **技术特点**：
  - 响应式系统：高效的数据绑定
  - 组合式API：逻辑复用性强
  - 类型检查：编译时错误检测

**Vite构建工具**
- **选择理由**：
  - 开发启动快，热更新迅速
  - 构建速度快，生产优化好
  - 配置简单，开箱即用

- **技术特点**：
  - ESM原生支持
  - 按需编译
  - 插件生态丰富

### 4.2.2 项目结构

