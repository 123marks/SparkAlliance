# 星火联盟 - 用户注册登录设计方案（优化版）

## 一、注册信息字段设计（优化后）

### 1.1 必填字段（核心信息）

```
1. 用户名/昵称（nickname）
   - 用途：用户在平台上的显示名称
   - 规则：
     * 长度：2-20个字符
     * 允许：中文、英文、数字、下划线
     * 不允许：特殊符号、空格、敏感词
     * 唯一性：不强制要求唯一（可重名），但建议唯一
   - 示例：星火少年、张三2024、study_master

2. 邮箱（email）
   - 用途：账号绑定、验证码登录、找回密码、重要通知
   - 规则：
     * 格式：标准邮箱格式
     * 正则：/^[^\s@]+@[^\s@]+\.[^\s@]+$/
     * 唯一性：必须唯一（一个邮箱只能注册一个账号）
   - 示例：user@example.com、student@university.edu.cn

3. 密码（password）
   - 用途：账号登录验证
   - 规则：
     * 长度：8-20个字符
     * 强度：必须包含字母和数字，建议包含特殊字符
     * 正则：/^(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d@$!%*#?&]{8,20}$/
     * 加密：使用bcrypt加密存储（cost factor: 10-12）
   - 示例：Password123!、Study@2024

4. 确认密码（confirmPassword）
   - 用途：防止用户输入错误
   - 规则：必须与password完全一致
```

### 1.2 选填字段（补充信息）

```
5. 手机号（phone）
   - 用途：账号绑定、找回密码、安全通知
   - 规则：
     * 格式：中国大陆手机号（11位数字）
     * 正则：/^1[3-9]\d{9}$/
     * 唯一性：如果填写，必须唯一
   - 示例：13812345678
   - 说明：选填，但建议填写，用于找回密码和重要通知

6. 学校（university）
   - 用途：学生认证、校园社区归属、个性化推荐
   - 规则：
     * 来源：从学校列表中选择（涵盖全国所有高校）
     * 必要性：建议填写，影响功能权限
   - 示例：清华大学、北京大学、浙江大学

7. 年级（grade）
   - 用途：用户画像、推荐算法、同龄人社交
   - 规则：从列表中选择
   - 选项：
     * 本科：大一、大二、大三、大四
     * 研究生：研一、研二、研三
     * 博士：博一、博二、博三、博四、博五
     * 其他：已毕业、教职工
   - 示例：大三、研一

8. 专业（major）
   - 用途：用户画像、推荐算法、专业社交
   - 规则：从列表中选择或自定义输入
   - 说明：涵盖教育部所有专业目录
   - 示例：计算机科学与技术、软件工程、电子信息工程
```

### 1.3 字段调整说明

**移除的字段：**
- ~~真实姓名（realName）~~：不需要，隐私保护
- ~~学号（studentId）~~：不需要，隐私保护

**调整原因：**
1. **隐私保护**：真实姓名和学号属于敏感隐私信息，不应在注册时收集
2. **简化流程**：减少用户注册阻力，提高注册转化率
3. **学校+年级+专业足够**：这三个字段已经能够很好地进行用户画像和个性化推荐
4. **学生认证可选**：如果后续需要学生认证，可以通过上传学生证等方式进行，不在注册时强制要求

### 1.4 系统自动生成字段

```
9. 用户ID（id）
   - 类型：UUID v4
   - 用途：用户唯一标识

10. 注册时间（createdAt）
    - 类型：Timestamp
    - 用途：记录注册时间

11. 最后登录时间（lastLoginAt）
    - 类型：Timestamp
    - 用途：记录最后登录时间

12. 账号状态（status）
    - 类型：Enum
    - 值：active（正常）、inactive（未激活）、banned（封禁）、deleted（已删除）

13. 认证状态（verificationStatus）
    - 类型：Object
    - 字段：
      * emailVerified: boolean（邮箱是否验证）
      * phoneVerified: boolean（手机是否验证，如果提供了手机号）
```

## 三、邮箱验证码方案（免费）

### 3.1 为什么选择邮箱验证码？

**优势：**
1. **免费**：自建SMTP服务器或使用免费额度，成本几乎为0
2. **安全**：邮箱验证同样安全，且邮箱账号通常更稳定
3. **正式**：适合学生群体，学生通常都有学校邮箱
4. **留存**：邮件可以保存，方便用户查看历史验证码

**劣势：**
1. **时效性**：邮件可能延迟（但通常几秒内到达）
2. **用户习惯**：部分用户可能不习惯查邮件

### 3.2 邮箱验证码流程

```
完整流程：

1. 用户输入邮箱
   ↓
2. 前端验证邮箱格式
   - 正则验证：/^[^\s@]+@[^\s@]+\.[^\s@]+$/
   - 不通过：提示"邮箱格式错误"
   ↓
3. 用户点击"发送验证码"按钮
   ↓
4. 前端调用后端API：POST /api/auth/send-email-code
   请求体：
   {
     "email": "user@example.com",
     "type": "register" // register, login, reset_password
   }
   ↓
5. 后端验证邮箱
   - 格式验证
   - 频率限制：同一邮箱60秒内只能发送1次
   - 次数限制：同一邮箱每天最多发送10次
   - 超限：返回错误"发送过于频繁，请稍后再试"
   ↓
6. 后端生成验证码
   - 6位随机数字：Math.random().toString().slice(2, 8)
   - 示例：123456
   ↓
7. 后端调用邮件服务发送验证码
   - 服务商：自建SMTP / SendGrid / Mailgun / 阿里云邮件推送
   - 模板：
     邮件标题：【星火联盟】验证码
     邮件内容：
       您好！
       您的验证码是：123456
       验证码5分钟内有效，请勿泄露给他人。
       如果这不是您的操作，请忽略此邮件。
       —— 星火联盟团队
   ↓
8. 后端存储验证码（Redis）
   - Key: email:code:type
   - Value: JSON.stringify({ code, attempts, createdAt })
   - TTL: 300秒（5分钟）
   ↓
9. 后端返回成功
   - 响应：{ success: true, message: "验证码已发送到您的邮箱" }
   ↓
10. 前端开始倒计时
    - 显示"60秒后重新发送"
    - 倒计时结束：显示"重新发送"
    - 提示用户查收邮件（包括垃圾邮件箱）
```

### 3.3 邮件服务商选择

| 服务商 | 免费额度 | 价格 | 到达率 | 推荐度 |
|--------|----------|------|--------|--------|

### 3.4 邮箱验证码安全机制

```
1. 频率限制
   - 同一邮箱：60秒内只能发送1次
   - 同一IP：1分钟内最多发送5次
   - 同一邮箱：每天最多发送10次

2. 验证码有效期
   - 默认：5分钟
   - 过期后：自动失效，需重新发送

3. 验证次数限制
   - 同一验证码：最多验证5次
   - 超过5次：验证码失效，需重新发送

4. 验证码复杂度
   - 长度：6位数字
   - 不使用简单序列：如123456、654321、111111等

5. 防爆破
   - 同一邮箱：连续验证失败5次，锁定30分钟
   - 同一IP：连续验证失败20次，锁定1小时

6. 日志记录
   - 记录所有发送和验证行为
   - 包括：邮箱、IP、时间、结果、原因
```

### 5.2 注册API设计

```javascript
/**
 * 用户注册API
 * POST /api/auth/register
 */
async function register(req, res) {
  try {
    const { nickname, email, password, phone, emailCode, university, grade, major } = req.body

    // 1. 验证邮箱验证码
    const emailRecord = await redis.get(`email:${email}:register`)
    if (!emailRecord) {
      return res.status(400).json({ error: '验证码已过期，请重新发送' })
    }
    
    const { code, attempts } = JSON.parse(emailRecord)
    if (attempts >= 5) {
      await redis.del(`email:${email}:register`)
      return res.status(400).json({ error: '验证次数过多，请重新发送' })
    }
    
    if (code !== emailCode) {
      await redis.set(`email:${email}:register`, JSON.stringify({ 
        code, 
        attempts: attempts + 1 
      }), 'EX', 300)
      return res.status(400).json({ error: '验证码错误' })
    }

    // 2. 验证邮箱是否已注册
    const existingUser = await db.query(
      'SELECT id FROM users WHERE email = $1',
      [email]
    )
    if (existingUser.rows.length > 0) {
      return res.status(400).json({ error: '该邮箱已注册' })
    }

    // 3. 验证手机号是否已注册（如果提供了手机号）
    if (phone) {
      const existingPhone = await db.query(
        'SELECT id FROM users WHERE phone = $1',
        [phone]
      )
      if (existingPhone.rows.length > 0) {
        return res.status(400).json({ error: '该手机号已注册' })
      }
    }

    // 4. 验证用户名敏感词
    if (containsSensitiveWords(nickname)) {
      return res.status(400).json({ error: '用户名包含敏感词' })
    }

    // 5. 验证密码强度
    if (!isStrongPassword(password)) {
      return res.status(400).json({ error: '密码强度不足' })
    }

    // 6. 加密密码
    const hashedPassword = await bcrypt.hash(password, 10)

    // 7. 生成用户ID
    const userId = uuidv4()

    // 8. 创建用户
    await db.query(`
      INSERT INTO users (
        id, nickname, email, password, phone, university, grade, major,
        email_verified, status, created_at, updated_at
      ) VALUES ($1, $2, $3, $4, $5, $6, $7, $8, true, 'active', NOW(), NOW())
    `, [userId, nickname, email, hashedPassword, phone, university, grade, major])

    // 9. 删除验证码
    await redis.del(`email:${email}:register`)

    // 10. 发送欢迎邮件
    await sendWelcomeEmail(email, nickname)

    // 11. 返回成功
    res.json({
      success: true,
      message: '注册成功',
      userId
    })

  } catch (error) {
    console.error('注册失败:', error)
    res.status(500).json({ error: '注册失败，请稍后再试' })
  }
}

const sendEmailCode = async () => {
  if (!isEmailValid.value) {

  }

  try {
    const response = await axios.post('/api/auth/send-email-code', {

    if (response.data.success) {

    errorMsg.value = '请输入6位验证码'

  }

    errorMsg.value = '密码长度应为8-20个字符'
    return
  }

  if (!/^(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d@$!%*#?&]{8,20}$/.test(form.value.password)) {
    errorMsg.value = '密码必须包含字母和数字'
    return
  }

  if (form.value.password !== form.value.confirmPassword) {
    errorMsg.value = '两次密码输入不一致'
    return
  }

  if (form.value.phone && !/^1[3-9]\d{9}$/.test(form.value.phone)) {
    errorMsg.value = '请输入正确的手机号'
    return
  }

    isLoading.value = false

## 九、总结

### 9.1 优化要点

1. **字段精简**：移除真实姓名、学号，保护隐私
2. **邮箱为主**：使用邮箱验证码（免费），手机号改为选填
3. **选项完善**：学校、年级、专业涵盖全国所有选项
4. **人机验证可选**：暂不使用，后期运营起来再添加
5. **用户体验**：减少注册步骤，提高转化率

### 9.2 成本分析

| 项目 | 方案 | 成本 | 说明 |
|------|------|------|------|
| 邮箱验证码 | SendGrid | 免费（100封/天） | 初期完全够用 |
| 人机验证 | 暂不使用 | 0元 | 后期添加 |
| 短信验证码 | 暂不使用 | 0元 | 手机号选填 |
| **总计** | - | **0元** | 初期完全免费 |

### 9.3 开发优先级

1. **Phase 1（当前）**：
   - 基础注册登录（邮箱+密码）
   - 邮箱验证码
   - 学校、年级、专业选择器

2. **Phase 2（后期）**：
   - 人机验证（运营起来后）
   - 手机号验证（如果需要）
   - 学生认证（如果需要）

### 9.4 技术选型

- **邮箱服务**：SendGrid（免费100封/天）
- **缓存**：Redis（存储验证码、频率限制）
- **密码加密**：bcrypt
- **Token**：JWT
- **人机验证**：暂不使用，后期添加
