# 个人主页设计分析与改进方案

## 一、问题分析：当前设计的错误

### 1.1 当前设计的问题

**错误理解：个人主页 ≠ 设置页面的个人资料**

```
当前错误设计：
用户点击"个人中心" → 跳转到设置页面的"个人资料"标签页

问题：
1. 概念混淆：个人主页是展示页面，设置是配置页面
2. 用户期望不符：用户想看自己的主页，不是修改资料
3. 功能缺失：缺少个人主页的核心功能（动态、商品、收藏等）
4. 体验割裂：无法快速查看自己的公开形象
```

### 1.2 正确的理解

**个人主页与设置的区别：**

| 维度 | 个人主页 | 设置页面 |
|------|---------|---------|
| **定位** | 展示页面 | 配置页面 |
| **目的** | 展示用户公开形象 | 修改账号配置 |
| **受众** | 自己 + 他人可见 | 仅自己可见 |
| **内容** | 动态、商品、收藏、关注等 | 个人资料、安全、隐私等 |
| **交互** | 浏览、互动、分享 | 编辑、配置、保存 |
| **入口** | 头像、昵称、"我的主页" | 设置图标、"设置" |

## 二、竞品分析：闲鱼与哔哩哔哩

### 2.1 闲鱼个人主页分析

**页面结构：**

```
闲鱼个人主页
├── 顶部信息区
│   ├── 用户头像（大）
│   ├── 用户昵称
│   ├── 认证信息（如实名认证、芝麻信用）
│   ├── 关注/粉丝数
│   ├── 个人简介
│   └── 操作按钮
│       ├── 关注（他人主页）
│       ├── 私信（他人主页）
│       └── 编辑资料（自己主页）
│
├── Tab导航区
│   ├── 出售中（商品列表）
│   ├── 已售出（交易记录）
│   ├── 点赞（收藏商品）
│   └── 关注（关注的人）
│
├── 内容展示区
│   ├── 商品卡片网格
│   │   ├── 商品图片
│   │   ├── 商品标题
│   │   ├── 价格
│   │   └── 状态标签
│   └── 空状态提示
│
└── 底部操作区
    ├── 发布商品按钮（悬浮）
    └── 分享主页
```

**核心特点：**

1. **展示优先**：主要展示用户的商品和交易信息
2. **Tab分类**：通过Tab清晰分类不同内容
3. **社交属性**：显示关注、粉丝、私信功能
4. **信任体系**：展示认证信息、信用评分
5. **快速操作**：悬浮按钮快速发布商品

**设计亮点：**

```
1. 信息层次清晰
   - 顶部：用户基本信息
   - 中部：Tab导航
   - 底部：内容展示

2. 内容丰富
   - 商品展示
   - 交易记录
   - 收藏夹
   - 关注列表

3. 交互便捷
   - 一键发布
   - 快速私信
   - 分享主页

4. 信任背书
   - 实名认证
   - 芝麻信用
   - 交易评价
```

## 五、总结

### 5.1 核心要点

1. **概念区分**：个人主页是展示页面，设置是配置页面
2. **学习竞品**：闲鱼、哔哩哔哩的设计模式
3. **内容为王**：以用户产生的内容为核心
4. **社交属性**：关注、粉丝、私信等社交功能
5. **数据展示**：清晰展示用户影响力和活跃度

### 5.2 设计原则

```
1. 展示优先
   - 主要展示用户的公开形象
   - 编辑功能作为次要入口

2. 内容导向
   - 以用户产生的内容为核心
   - 动态、商品、收藏等

3. 社交互动
   - 关注、粉丝、私信
   - 分享、互动

4. 数据支撑
   - 关注/粉丝数
   - 获赞数
   - 活跃度

5. 信任背书
   - 认证标识
   - 等级徽章
   - 信用评分
```

### 5.3 实现优先级

```
Phase 1（核心功能）：
- 顶部信息区
- 数据统计区
```

```sql
CREATE TABLE IF NOT EXISTS follows (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  follower_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,  -- 关注者
  following_id UUID REFERENCES auth.users(id) ON DELETE CASCADE, -- 被关注者
  created_at TIMESTAMPTZ DEFAULT NOW(),
  
  -- 唯一约束：同一对关系只能存在一次
  UNIQUE(follower_id, following_id),
  
  -- 检查约束：不能关注自己
  CHECK (follower_id != following_id)
);

-- 索引优化
CREATE INDEX idx_follows_follower ON follows(follower_id);
CREATE INDEX idx_follows_following ON follows(following_id);

-- RLS 策略
ALTER TABLE follows ENABLE ROW LEVEL SECURITY;
CREATE POLICY "follows_select" ON follows FOR SELECT USING (true);
CREATE POLICY "follows_insert" ON follows FOR INSERT 
  WITH CHECK (auth.uid() = follower_id);
CREATE POLICY "follows_delete" ON follows FOR DELETE
```

### 10.2 数据共享策略

```typescript
// composables/useProfile.ts
import { ref, readonly } from 'vue'

// 全局状态（缓存用户资料）
const profileCache = new Map<string, any>()

export function useProfile() {
  const getProfile = async (userId: string, forceRefresh = false) => {
    // 命中缓存且不强制刷新
    if (!forceRefresh && profileCache.has(userId)) {
      return profileCache.get(userId)
    }
    
    const profile = await fetchProfileFromDB(userId)
    profileCache.set(userId, profile)
    return profile
  }
  
  const refreshProfile = (userId: string) => {
    return getProfile(userId, true)
  }
  
  const clearCache = () => {
    profileCache.clear()
  }
  
  return {
    getProfile,
    refreshProfile,
    clearCache
  }
}
```

### 11.1 隐私保护

**数据可见性控制：**
```typescript
// 用户可控制的隐私设置
interface PrivacySettings {
  show_online_status: boolean    // 显示在线状态
  show_school_info: boolean      // 显示学校信息
  show_favorites: boolean        // 显示收藏列表
  allow_stranger_dm: boolean     // 允许陌生人私信
  allow_search_by_id: boolean    // 允许通过SparkID搜索
  profile_visibility: 'public' | 'followers' | 'private'  // 主页可见性
}

// API 层面过滤
async function getProfileWithPrivacy(userId: string, viewerId: string) {
  const profile = await getProfile(userId)
  const privacy = await getPrivacySettings(userId)
  
  // 非自己访问时，根据隐私设置过滤数据
  if (userId !== viewerId) {
    if (!privacy.show_school_info) {
      delete profile.college
      delete profile.major
    }
    if (privacy.profile_visibility === 'private') {
      // 私密账号，只显示基本信息
      return { id: profile.id, nickname: profile.nickname, avatar_url: profile.avatar_url }
    }
  }
  
  return profile
}
```
