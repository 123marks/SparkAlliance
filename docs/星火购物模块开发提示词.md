# 星火购物模块开发提示词

## 一、项目背景与定位

### 1.1 产品定位
星火购物是一个面向高校校园的二手交易与闲置物品流转平台，类似于闲鱼的C2C模式，但专注于校园场景。核心价值：
- **信任基础**：基于校园身份认证，交易双方都是本校学生，降低信任成本
- **便捷交易**：校内面交为主，无需物流，交易更快捷
- **闲置流转**：帮助学生处理闲置物品，实现资源循环利用
- **社交属性**：通过交易建立同学间的联系，增强社区氛围

### 1.2 目标用户
- **主要用户**：在校大学生（本科生、研究生）
- **次要用户**：教职工、校友
- **用户特征**：
  - 年龄：18-30岁
  - 需求：购买二手教材、电子产品、生活用品；出售闲置物品
  - 痛点：校外平台信任成本高、物流麻烦、价格不透明

## 二、核心功能模块

### 2.1 商品发布模块
**功能描述**：用户发布闲置物品信息

**核心字段**：
```
- 商品标题（必填，5-50字）
- 商品描述（必填，20-500字）
- 商品分类（必选，支持多级分类）
  * 教材书籍：教材、课外书、考研资料、考证资料
  * 电子产品：手机、电脑、平板、耳机、相机、配件
  * 生活用品：日用品、服饰、美妆、运动健身
  * 学习用品：文具、笔记本、打印资料
  * 其他：其他闲置物品
- 商品成色（必选）：全新、几乎全新、轻微使用痕迹、明显使用痕迹
- 商品价格（必填，支持面议）
- 原价（选填，用于展示折扣力度）
- 商品图片（必填，1-9张，首图作为封面）
- 交易方式（必选）：校内面交、宿舍楼交易、约定地点
- 交易地点（选填）：具体交易地点描述
- 是否可议价（默认可议价）
- 商品标签（选填，如：急出、包邮、可小刀等）
```

**UI设计要点**：
- 参考：闲鱼发布页面的简洁设计
- 图片上传：支持拖拽、粘贴、批量上传，自动压缩
- 智能识别：集成AI识别商品类型（可选）
- 价格建议：根据商品类型和历史数据给出价格建议
- 草稿保存：自动保存草稿，防止数据丢失

**交互细节**：
- 实时字数统计
- 图片预览与排序
- 分类快速选择（常用分类置顶）
- 发布前预览功能
- 敏感词过滤提示

### 2.4 交易管理模块

**4.1 交易流程**
```
标准流程：
1. 买家发起"我想要"或聊天
2. 双方沟通议价
3. 约定交易时间、地点
4. 线下见面交易
5. 买家确认收货
6. 交易完成

状态流转：
- 待交易（双方约定中）
- 交易中（已约定时间地点）
- 已完成（买家确认收货）
- 已取消（任一方取消）
```

**4.2 订单管理**
```
买家订单：
- 我想要的（收藏）
- 进行中的交易
- 已完成的交易
- 已取消的交易

卖家订单：
- 我发布的
- 待处理（有人想要）
- 进行中
- 已完成
- 已取消
```

**4.3 交易评价**
```
评价维度：
- 交易体验评分（1-5星）
- 描述相符度
- 沟通态度
- 评价内容（选填，20-200字）
- 评价图片（选填，最多3张）

评价展示：
- 用户信用分 = 历史评价综合计算
- 评价列表展示在用户主页
```

## 三、安全与风控机制（重点）

### 3.1 身份认证体系
```
强制认证：
- 学生认证：必须上传学生证/校园卡，经审核通过后才能发布商品
- 手机认证：必须绑定手机号

可选认证：
- 实名认证：提升信用分
- 人脸认证：高价值商品交易时验证
```

### 3.2 内容审核机制
```
自动审核（AI）：
- 图片审核：识别违规图片（涉黄、涉暴、广告等）
- 文字审核：敏感词过滤、广告识别
- 价格审核：异常价格检测（如：iPhone卖1元）

人工审核：
- 用户举报内容
- AI标记的可疑内容
- 高价值商品（价格>1000元）
```

### 3.3 交易风控
```
风险提示：
- 引导用户使用平台内聊天，不要加微信/QQ
- 提醒用户当面验货，贵重物品建议录像
- 警惕异常低价、虚假宣传

异常检测：
- 同一用户频繁发布/取消
- 异常价格波动
- 疑似诈骗行为（如：要求先转账）
- 刷单行为检测

处罚机制：
- 违规商品：下架、删除
- 违规用户：警告、禁言、封号
- 严重违规：永久封禁、报警
```

### 3.4 资金安全
```
建议方案：
- 初期：纯线下交易，平台不涉及资金流转
- 中期：引入担保交易（类似闲鱼）
  * 买家付款到平台
  * 买家确认收货后，平台打款给卖家
  * 纠纷时平台介入仲裁
- 后期：引入支付分、信用体系
```

### 3.5 隐私保护
```
数据保护：
- 手机号脱敏展示
- 学生证照片加密存储
- 交易记录仅双方可见

权限控制：
- 用户可设置是否公开在售商品
- 用户可设置是否允许陌生人聊天
- 用户可拉黑其他用户
```

```
AI审核：阿里云内容安全 / 腾讯云天御
```

### 4.3 数据库设计（核心表）

**商品表（products）**
```sql
CREATE TABLE products (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  seller_id UUID REFERENCES users(id) NOT NULL,
  title VARCHAR(100) NOT NULL,
  description TEXT NOT NULL,
  category_id INTEGER REFERENCES categories(id),
  condition VARCHAR(20) NOT NULL, -- 'new', 'like_new', 'good', 'fair'
  price DECIMAL(10, 2) NOT NULL,
  original_price DECIMAL(10, 2),
  images TEXT[] NOT NULL, -- 图片URL数组
  trade_method VARCHAR(50) NOT NULL,
  trade_location VARCHAR(200),
  is_negotiable BOOLEAN DEFAULT true,
  tags TEXT[],
  status VARCHAR(20) DEFAULT 'active', -- 'active', 'sold', 'offline', 'deleted'
  view_count INTEGER DEFAULT 0,
  want_count INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  sold_at TIMESTAMP,
  deleted_at TIMESTAMP
);

-- 索引
CREATE INDEX idx_products_seller ON products(seller_id);
CREATE INDEX idx_products_category ON products(category_id);
CREATE INDEX idx_products_status ON products(status);
CREATE INDEX idx_products_created ON products(created_at DESC);
CREATE INDEX idx_products_price ON products(price);
```

**分类表（categories）**
```sql
CREATE TABLE categories (
  id SERIAL PRIMARY KEY,
  name VARCHAR(50) NOT NULL,
  parent_id INTEGER REFERENCES categories(id),
  icon VARCHAR(200),
  sort_order INTEGER DEFAULT 0,
  is_active BOOLEAN DEFAULT true
);
```

**交易表（transactions）**
```sql
CREATE TABLE transactions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  product_id UUID REFERENCES products(id) NOT NULL,
  buyer_id UUID REFERENCES users(id) NOT NULL,
  seller_id UUID REFERENCES users(id) NOT NULL,
  status VARCHAR(20) DEFAULT 'pending', -- 'pending', 'in_progress', 'completed', 'cancelled'
  agreed_price DECIMAL(10, 2),
  trade_time TIMESTAMP,
  trade_location VARCHAR(200),
  buyer_confirmed BOOLEAN DEFAULT false,
  seller_confirmed BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT NOW(),
  completed_at TIMESTAMP,
  cancelled_at TIMESTAMP,
  cancel_reason TEXT
);
```

**评价表（reviews）**
```sql
CREATE TABLE reviews (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  transaction_id UUID REFERENCES transactions(id) NOT NULL,
  reviewer_id UUID REFERENCES users(id) NOT NULL,
  reviewee_id UUID REFERENCES users(id) NOT NULL,
  rating INTEGER CHECK (rating >= 1 AND rating <= 5),
  description_match INTEGER CHECK (description_match >= 1 AND description_match <= 5),
  attitude INTEGER CHECK (attitude >= 1 AND attitude <= 5),
  content TEXT,
  images TEXT[],
  created_at TIMESTAMP DEFAULT NOW()
);
```

**聊天消息表（messages）**
```sql
CREATE TABLE messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  conversation_id UUID NOT NULL,
  sender_id UUID REFERENCES users(id) NOT NULL,
  receiver_id UUID REFERENCES users(id) NOT NULL,
  content TEXT,
  message_type VARCHAR(20) DEFAULT 'text', -- 'text', 'image', 'product', 'system'
  product_id UUID REFERENCES products(id),
  is_read BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_messages_conversation ON messages(conversation_id, created_at DESC);
```

**用户认证表（user_verifications）**
```sql
CREATE TABLE user_verifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) NOT NULL,
  verification_type VARCHAR(20) NOT NULL, -- 'student', 'real_name', 'phone'
  status VARCHAR(20) DEFAULT 'pending', -- 'pending', 'approved', 'rejected'
  document_url TEXT, -- 证件照片URL
  verified_data JSONB, -- OCR识别结果
  reviewer_id UUID REFERENCES users(id),
  review_note TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  reviewed_at TIMESTAMP
);
```

## 七、关键开发提示词

### 7.1 给前端开发者的提示词

```
你正在开发一个校园二手交易平台的前端部分，参考闲鱼的设计风格。

核心要求：
1. 使用 Vue 3 + TypeScript + Vite 技术栈
2. UI风格：简洁、年轻、活力，主色调蓝色#4F8EF7
3. 移动端优先，响应式设计

关键页面：
1. 首页：搜索栏 + 分类导航 + 商品瀑布流（双列布局）
2. 发布页：图片上传 + 表单填写 + 预览
3. 商品详情页：图片轮播 + 基本信息 + 卖家信息 + 操作按钮
4. 聊天页：聊天列表 + 聊天详情 + 商品卡片嵌入
5. 用户中心：个人信息 + 认证 + 我的商品 + 我的交易

技术要点：
- 图片上传：支持拖拽、粘贴、批量上传，自动压缩
- 瀑布流布局：使用CSS Grid或第三方库
- 实时通讯：WebSocket连接，消息已读/未读状态
- 无限滚动：使用Intersection Observer API
- 图片懒加载：使用Intersection Observer或第三方库

安全注意：
- 所有用户输入必须做XSS过滤
- 图片上传前检查文件类型和大小
- 敏感信息（手机号）脱敏展示
- 不要在localStorage存储敏感信息

请先实现MVP版本，包括：首页、发布页、商品详情页、聊天页、用户中心。
```

---

### 7.2 给后端开发者的提示词

```
你正在开发一个校园二手交易平台的后端API，核心是安全、稳定、高性能。

核心要求：
1. 使用 Node.js + Express/NestJS + PostgreSQL + Redis
2. 所有API必须做身份验证（JWT）
3. 所有用户输入必须做验证和过滤
4. 关键操作必须记录日志

核心API设计：

1. 用户认证相关
POST /api/auth/register - 注册
POST /api/auth/login - 登录
POST /api/auth/logout - 登出
```
