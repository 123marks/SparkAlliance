## 🎯 核心规则

### 规则1：文件管理规则

**问题描述：**
- 每次新建文件而不删除旧文件
- 导致文件冗余，影响效率和质量

**解决方案：**
```
同一性质的文件更新时，直接覆盖原文件，不创建新文件
```

**具体规则：**

1. **文档更新规则**
   - 如果是同一主题的文档更新，直接覆盖原文件
   - 不创建类似 `xxx-v2.md`、`xxx-new.md` 这样的文件
   - 保持文件名简洁统一

2. **示例：**
   ```
   ❌ 错误做法：
   - CODEX-CORRECT-USAGE.md
   - CODEX-MAIN-BRAIN.md
   - CODEX-GUIDE.md（新建）
   
   ✅ 正确做法：
   - CODEX-GUIDE.md（直接覆盖，合并内容）
   - 删除旧的重复文件
   ```

3. **判断标准：**
   - 文件主题相同 → 覆盖原文件
   - 文件主题不同 → 创建新文件
   - 不确定时 → 询问用户

---

### 规则2：文档命名规范

**命名原则：**
- 简洁明了
- 一目了然
- 避免冗余

**标准命名：**
```
✅ SparkAlliance.md      - 项目骨架文档
✅ AGENTS.md             - 开发进度追踪
✅ CODEX-GUIDE.md        - Codex使用指南
✅ CLI-COMMANDS.md       - CLI命令参考
✅ PROJECT-RULES.md      - 项目全局规则（本文档）
```

**避免命名：**
```
❌ SparkAlliance-v2.md
❌ SparkAlliance-new.md
❌ SparkAlliance-final.md
❌ SparkAlliance-20260315.md
```

---

### 规则3：内容更新规则

**更新原则：**
- 增量更新，不重复创建
- 合并相似内容
- 保持文档简洁

**具体规则：**

1. **新增内容**
   - 直接在原文件中添加新内容
   - 不创建新文件

2. **修改内容**
   - 直接修改原文件
   - 不创建备份文件

3. **删除内容**
   - 直接删除不需要的内容
   - 不保留历史版本（使用Git管理历史）

## 📝 文档更新规则

### 规则8：文档更新流程

**标准流程：**
```
1. 确定要更新的文档
2. 读取现有内容
3. 合并新内容
4. 覆盖原文件
5. 删除冗余文件
```

**示例：**
```python
# 伪代码示例
def update_document(topic, new_content):
    # 1. 查找同主题的现有文档
    existing_doc = find_document_by_topic(topic)
    
    if existing_doc:
        # 2. 合并内容
        merged_content = merge(existing_doc.content, new_content)
        # 3. 覆盖原文件
        overwrite(existing_doc.path, merged_content)
    else:
        # 4. 创建新文件
        create_new_document(topic, new_content)
    
    # 5. 清理重复文件
    cleanup_duplicate_files()
```

---

### 规则9：内容合并原则

**合并原则：**
1. **保留最新信息** - 优先保留最新的内容
2. **去重** - 删除重复的内容
3. **结构化** - 保持良好的文档结构
4. **完整性** - 确保信息完整不丢失

## 🎯 特殊场景处理

### 场景1：不确定是否应该覆盖

**处理方式：**
```
1. 先询问用户
2. 说明两个文件的内容
3. 让用户决定是否合并
```

---

### 场景2：文件内容冲突

**处理方式：**
```
1. 保留更详细的内容
2. 合并不冲突的部分
3. 标注冲突部分让用户决定
```

---

### 场景3：历史版本需要保留

**处理方式：**
```
1. 使用Git管理历史版本
2. 不创建备份文件
3. 通过Git回溯历史
```

## 🎯 总结

**核心原则：**
1. ✅ 同主题内容合并到一个文件
2. ✅ 直接覆盖原文件，不创建新文件
3. ✅ 保持文件命名简洁统一
4. ✅ 定期清理冗余文件
5. ✅ 使用Git管理历史版本

**禁止行为：**
1. ❌ 创建 `xxx-v2.md`、`xxx-new.md` 等文件
2. ❌ 保留多个同主题的文件
3. ❌ 不删除临时文件
4. ❌ 文件命名不规范

**目标：**
- 保持项目文件结构清晰
