## 📝 代码规范

### 代码风格
- **简洁干净：** 避免冗余代码
- **可读性强：** 变量名要有意义
- **模块化：** 功能拆分清晰
- **注释完整：** 关键逻辑必须注释

### 命名规范
```javascript
// ✅ 好的命名
const getUserInfo = () => {}
const handleSubmit = () => {}
const userList = []

// ❌ 不好的命名
const a = () => {}
const func1 = () => {}
const list = []
```

### 代码格式
```javascript
// ✅ 好的格式
function calculateTotal(items) {
  // 计算总价
  let total = 0;
  items.forEach(item => {
    total += item.price * item.quantity;
  });
  return total;
}

// ❌ 不好的格式
function calculateTotal(items){let total=0;items.forEach(item=>{total+=item.price*item.quantity});return total;}
```

### 提交时机

- 代码审查通过后
- 优化完成后

```

fix: 修复登录bug
docs: 更新API文档
test: 添加单元测试
```

```
何时回滚：
- 代码审查评分 < 7分

- 需要重新设计

如何回滚：
git reset --soft HEAD~1
```
